---
title: "MCP Server"
---

The SessionFS MCP server lets AI agents search your past sessions and read project context during conversations. Runs on stdio transport for local use, or HTTP/SSE for remote access.

## Install for Claude Code

```bash
sfs mcp install --for claude-code
```

This adds SessionFS to your Claude Code MCP configuration automatically.

## Install for Cursor

```bash
sfs mcp install --for cursor
```

## Available tools

The MCP server exposes 12 tools:

| Tool | Description |
|------|-------------|
| `search_sessions` | Search past sessions by keyword, error message, or file path |
| `get_session_context` | Get full conversation context from a specific session |
| `list_recent_sessions` | List recent sessions with filtering |
| `find_related_sessions` | Find sessions related to specific files or errors |
| `get_project_context` | Get shared project context document for a repository (includes wiki pages, pending entries, contribution instructions) |
| `get_session_summary` | Get the deterministic or narrative summary for a session |
| `get_audit_report` | Get the LLM Judge audit report for a session |
| `add_knowledge` | Add a knowledge entry to the project (6 types: decision, convention, pattern, dependency, issue, note) |
| `update_wiki_page` | Create or update a wiki page in the project knowledge base |
| `list_wiki_pages` | List all wiki pages for a project |
| `search_project_knowledge` | Search across knowledge entries and wiki pages |
| `ask_project` | Ask a question about the project using compiled knowledge |

## How agents use it

When connected, AI agents can:
- Search for past solutions: "Have we solved this auth error before?"
- Load full session context: "Show me the session where we debugged the migration"
- Read project conventions: "What are the team's coding standards?"

## Manual start

```bash
sfs mcp serve
```

Runs on stdio transport. Connect from any MCP-aware tool.
